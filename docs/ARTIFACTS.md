# Artifact Policy

Large model files are intentionally kept out of normal source-control changes
going forward. This keeps the project easy to clone and makes the source tree
focus on reproducible code.

## Expected Local Layout

Place model artifacts here when running inference:

```text
artifacts/
  best_cnn_model.pth
  cnn_thresholds.pt
```

These files are generated by `scripts/train.py` or can be downloaded from the
initial artifact release:

```bash
mkdir -p artifacts
gh release download visiontagger-v0.1-artifacts \
  --repo nicholas-tayag/pytorch-multilabel-vision \
  --dir artifacts
```

## Current Artifact Roles

- `best_cnn_model.pth`: best validation checkpoint for the transfer-learning
  ResNet-18 model.
- `cnn_thresholds.pt`: per-class validation thresholds used during prediction.
- `baseline_resnet18_scratch.pth`: scratch baseline checkpoint, useful for
  comparison but not required for inference.
- `ensemble_model.pth`: averaged checkpoint experiment, useful as a stretch
  comparison.

## Recreating Artifacts

```bash
uv run python scripts/train.py --config configs/default.yaml
```

The configured output directory defaults to:

```text
runs/vision_tagger/
```
